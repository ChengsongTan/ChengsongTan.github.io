Suggestion 1

Section 3.2.4.2.4 CXL.cache Write says:

3.2.4.2.4 CLFlush

... The Host considers a write to be done once it has received all 64 bytes of data, and has sent the GO-I response message. ...
This leaves the impression that GO and the pulled Data always come in pairs. However, there is one exception where CleanEvictNoData does not require a piece of Data to be returned. In that case, just the GO-I being sent would suffice as a sign of transaction completeness for the Host.

Tentative suggestion

We propose the following text to be added to avoid any confusion about CleanEvictNoData's transaction flows (and when it was considered complete):

Section 3.2.4.2.4 CXL.cache Write

... The Host considers a write to be done once it has received all 64 bytes of data, and has sent the GO-I response message. In the case where only a GO-I is required (i.e. CleanEvictNoData), just the GO message being sent would be enough for the Host to consider a write done. ...
